Getting Started

Before starting to use this source code and developing app you need to first install the following softwares to your computer. If you have already installed these softwares you can skip this step.

1. JDK 5 or above: Install Java Development Kit (JDK) 5 or greater. You can download here.

2. Eclipse IDE: You can choose Eclipse Indigo or other packages. I prefer Indigo. You can download here. Extract the zip file into a folder for example E:\ANDROID\Eclipse after downloading. Run eclipse.exe. On first time you will be asked for workspace directory. It is a directory where you want your android project to be stored. For now just create new folder Workspace in E:\ANDROID or any other directory as you wish.

3. Android SDK: Install android sdk. You can download here. Now you need to connect Eclipse and Android sdk. On Eclipse GUI, select Windows > Preferences menu on top. Select Android and then locate your SDK Location, on my machine it is G:\EclipseAndSDK\AndroidSDK4.0, and then click Apply.

AdMob SDK 4.3.1 and upwards require Android 3.2 (API 13) or above for compilation. So make sure you have downloaded till Android 3.2 SDK. Though you need to compile with Android 3.2 + and the application works perfectly in devices having Android 2.2 and higher.

Importing Project into Eclipse

1. Extract the downloaded archive that you have purchased.

2. Open Eclipse > Select File > Import.

3. In the Import window page select Existing Projects into Workspace under General and click Next. In the next window choose Select root directory. Press the Browse button and choose SourceCodeProject folder present in the Project folder. Press Enter and Select/Check "android-support-v7-appcompat" and "AndroidWebsiteTemplate". Click Finish.

4. Your project will be visible on the Package Explorer. At the left side you can see the project folder. You can edit required files for default colors and your website urls.
Replace the colors and urls in AppConfig.java file as shown in below screenshot.

Changing Package Name

You need to change package name in the .java files (src folder), .xml(res-layout) and AndroidManifest.xml.

Replace in java files, res-layout .xml files and AndroidManifest.xml: Open all the files in the src folder, res-layout folder and AndroidManifest.xml at once in Notepad++. Select Find/Replace option and replace from "sale.app.livewallpaper.gif" to "your.package.name". 

Do not select "Match case".

You may find it little hard to replace it in first try if you have not used Notepad++ before, but in few attempts you can do it. For experienced users this is a simple task.

Changing App Name

App Name: Go to res/values/ directory and edit strings.xml file.

To change App name: Enter the name in the below string.

<string name="app_name">Gif Live Wallpaper</string>

Changing Application Icon

The application icon is present at res/drawable-{dpi}/ic_launcher.png. Create your icon image with the same file name and replace the existing file.

This takes different sizes icons for different drawable folders:
drawable-xhdpi 96 * 96
drawable-hdpi 72 * 72
drawable-mdpi 48 * 48
drawable-ldpi 36 * 36

Refresh the project for changes to take effect, clean and Run.

Enabling AdMob Ads

GIF Live Wallpaper comes with AdMob Banner ads and Interstitial ads enabled. You can easily enable/disable the ads (true - enabled, false - disabled) in design.website.template.config.AppConfig.java file, in place of below line value:

public static final boolean BANNER_ADS_ENABLED = true;
public static final boolean INTERSTITIAL_ADS_ENABLED = true;

You need to provide admob ads unit id in the same file in place of "PUT_YOUR_ADMOB_ID_HERE" in below line, otherwise ads will not pop up on screen:

public static final String ADMOB_ADS_ID = "PUT_YOUR_ADMOB_ID_HERE";

Customization & Theme

Replace GIF file
You need to replace project .gif file in res/raw/gif_image.gif to your liking .gif file and keep the same name like "gif_image.gif".
The look, feel, performance and smoothness depends on .gif file.

Theme
You can easily customize this app to your liking colors like screen bg color, button and message text color, bottom bar color, button normal and selected/hover bg color and icons selected color. You can change colors in res/values/colors.xml file, in place of below lines color values:

<color name="screen_bg_color">#648609</color>
<color name="button_text_color">#ffffff</color>
<color name="message_text_color">#424242</color>
<color name="bottom_bar_color">#AAFFFFFF</color>
<color name="button_border_color">#0006ff</color>
<color name="button_normal_color">#ffa200</color>
<color name="button_selected_color">#80ac0b</color>
<color name="icons_selector_color">#AAFFFFFF</color>

More and Share App Icons
You can easily change menu icons default values (true - enabled, false - disabled), in design.website.template.config.AppConfig.java file, in place of below lines values:

public static final boolean MORE_APP_ENABLED = true;
public static final boolean SHARE_APP_ENABLED = true;

You need to provide one more value for More Apps menu option to show your PlayStore apps page, in design.website.template.config.AppConfig.java file, in place of below lines values:

public static final String PLAYSTORE_ACCOUNT_NAME = "PUT_YOUR_PLAYSTORE_ACCOUNT_NAME_HERE";

MOST IMPORTANT
Testing Flag : You need to make this flag to "false" before publishing this app. This flag is used to send "testing" for AdMob ads. Don't forget to update the flag to "false" in below line:

public static final boolean TESTING_ENABLED = true;

Compiling the Application

After customizing this app, you need to compile into an .apk file that can be installed in Android devices. Here are the steps of compiling this application.

1. Select the GIFLiveWallpaper project in Package Explorer, Right click and select Export.

2. In Export window select Android > Export Android Application. Then, click Next button.

3. Your project name will appear in Project: box. If not, click Browse and find. Click Next button to proceed.

4. Select your keystore or create a new one. In Location, click Browse button to decide keystore name and the location of it. Give a strong password and proceed to next step.

5. Fill the key creation form and press Next.

6. Finally, click Browse button to specify the location where you want to have the generated apk, and then click Finish. Now your application is ready. Install it. Have fun.
